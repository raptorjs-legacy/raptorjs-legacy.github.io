Frequently Asked Questions

Why can't I just downloaded the JavaScript code for RaptorJS?
Downloading pre-optimized and bundled builds of RaptorJS goes against the modularity philosophy of RaptorJS. The RaptorJS Optimizer makes it easy to pull in only the dependencies that your web page needs and that tool should be used to pull in JavaScript and CSS code into your web application.
How do I install RaptorJS packages into my application?

RaptorJS does not currently include a package manager that handles the distribution of RaptorJS packages. Therefore, you will need to use your own scripts or tools into "install" a RaptorJS package into your application. RaptorJS allows developers to any directory to be added to the resource search path so simply add directories that contain the installed RaptorJS packages to the RaptorJS resource search path.

Node includes npm which is a package manager and that package manager can be used to install packages into the node_modules directory. You can use your own scripts to move RaptorJS modules into their own directory or simply add the node_modules directory to the RaptorJS resource search path.

Another option is to use Twitter Bower which is an "unopinionated" package manager that allows you to install packages from Git repositories and URLs into your project.

How do I share RaptorJS packages with others?

Since RaptorJS does not currently include a package manager, we recommend using either Twitter Bower or npm to share RaptorJS packages.
